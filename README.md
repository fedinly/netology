# ЗАДАЧА 2
## Вопрос 1
    Для высоконагруженной базы данных MySQL, критичной к отказц, можно выбрать варианты с физическими серверами и осуществлять резервирование средствами БД (репликация) или с виртуализацией уровня ОС и резервировать ВМ.
## Вопрос 2
    Для различных web-приложений подойдет паравиртуализация или виртуализация уровня ОС (быстрое развертывание, масштабирование).
## Вопрос 3
    Windows-системы для использования бкхгалтерским отделом могут быть установлены на физические серверы (файловые БД 1С, файловые шары, принтеры и т.п.). Резервное копирование может быть осуществлено средствами ОС, при выходе из строя сервера БД может быть перенесена на соседний ПК в сети.
## Вопрос 4
    Системы, выполняющие высокопроизводительные расчеты на GPU могут быть установлены как на физ.сервер, так и на ВМ уровня ОС (для ускорения доступа к ресурсам GPU). Также на физ.ПК можно установить большее кол-во видеокарт для работы приложений. 
# ЗАДАЧА 3
## Вопрос 1
    Для данного сценария с учетом требования наличия автоматического механизма создания рез.копий и разнообразия требуемых ОС можно порекомендовать облачную платформу (напр., Yandex Cloud).
## Вопрос 2
     Наиболее подходит решение Proxmox VE (виртуализация уровня ОС, opensource, поддерживает разные гостевые ОС) 
## Вопрос 3
     Подходящим решением для виртуализации windows-based инфраструктуры может стать Hyper-V Core на основе Windows Server Core. Он бесплатный, с хорошей совместимостью и производительностью.
